# Websocket协议之基础

# 写在前面

本文是说websocket的基础知识，其原理，其特点和使用场景，但websocket属于比较新的协议，所以本文是建立在你已经有相关的计算机网络知识基础来看，本文也会一直用简单明了的比喻和大白话来形容websocket。

# 那啥是websocket？

说句实话，在毕业找工作的开始我都不知道这个东西，也没有实际的使用过，当面试官问：“你简单说说你对websocket的理解”，我一脸懵逼，当时就深恶痛绝要弄懂这个，后来又自己看了nodeJS弄了一些简单的聊天室呀之类的，那我们回归正题：

老规矩，让我们看一看官网上的“文言文”版定义是啥？

WebSocket是一种通信协议，可在单个TCP连接上进行全双工通信。WebSocket使得客户端和服务器之间的数据交换变得更加简单，允许服务端主动向客户端推送数据。在WebSocket API中，浏览器和服务器只需要完成一次握手，两者之间就可以建立持久性的连接，并进行双向数据传输。

其实针对这句话来说，重点的话就那么几句：建立在TCP连接之上、允许服务端主动向客户端推送数据的双向通信、需要完成一次握手（不是直接就能连上的，得握手）

看完上面的内容，我就做一下对官方定义的白话翻译：

Websocket白话翻译

WebSocket就是和http他们一样的一种网络通信协议，建立在一个TCP上。客户端和服务器通过TCP完成一次握手之后，两者就能互相发数据（重点是服务器也能给客户端发了），所以双方的连接生命周期更长，是持久的连接，通过这些方式实现了双向数据传输，让两者的数据交换变得更加简单。

# websocket还有哪些特点？

除了我们上面说的那些之外，还有一些其他的优化特点：

  • 握手阶段采用 HTTP 协议。
  • 数据格式轻量，性能开销小。客户端与服务端进行数据交换时，服务端到客户端的数据包头只有2到10字节，客户端到服务端需要加上另外4字节的掩码。HTTP每次都需要携带完整头部。
  • 更好的二进制支持，可以发送文本，和二进制数据
  • 没有同源限制，客户端可以与任意服务器通信
  • 协议标识符是ws（如果加密，则是wss），请求的地址就是后端支持websocket的API（服务器网址就是 URL）。
