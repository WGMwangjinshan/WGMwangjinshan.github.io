动态路由（addRoutes）

王金山 2020-08-17 Vue

动态路由是啥？

从这个名字来看，动态路由是一个可以动态变化的路由结构，不同于以往的静态路由，基本上页面的结构就是那个样子，不会有改变。动态路由的页面结构和跳转结构是变化的，所以需要动态路由。

动态路由是干什么的？

目前我接触到的动态路由的使用是来实现权限。我们来想一个权限可以体现在什么地方？可能是用户是否能看见，可能是用户是否可以进行跳转此操作。上面我们讨论的是颗粒度为==页面级别==的权限，我认为基本上可以从路由守卫或者是meta里的权限控制字段来实现这个权限控制。

但是假设我们有一个需求是页面的导航都会随时在变的，例如我现在有两个一级导航叫做：质数和奇数，那我们现在有一个二级导航叫做“1”，那么这个叫“1”的导航你说是在质数导航下也可以，你说他在奇数导航上也可以，这个就很因人而异。所以可能会导致不同用户对导航的需求也不尽相同，那这个时候写死的导航或者通过字段或者路由守卫就实现不了这种权限的设置，就一定要动态的添加路由才可以。

==我们可以看到上面的结构和静态路由的结构也是一样的，只不过是后台返回给我们的，这才是动态==

在这个地方我们注意，后台传给我们的数据有一个关键的字段“component”，按我们正常的注册路由方式，这个地方应该这么写：() => import('@/views/login/index')，但是这里是字符串“login”，所以我们要自己对这个字符串进行赋值修改。

==这个地方我们使用map来进行循环赋值，下面是例子：==

var map = new Map()
map.set("login", () => import('@/views/login/index'))
map.set("register", () => import('@/views/register/register'))//注册
map.set("forgetPassword", () => import('@/views/forgetPassword/forgetPassword'))//忘记密码
map.set("error", () => import('@/views/error'))
map.set("home", () => import('@/views/home/index'))
map.set("index", () => import('@/views/home/index/index'))

        userRouter.forEach(function(item, index) {

== ！！！！！注意！！！！！：上面的所有代码在项目中都是一个方法，在页面开始渲染的时候不可能直接把所有的路由就配置好，因为路由有一部分是从后台动态获取的。所以这个地方需要我们调用方法来实现对路由的初始化，之后addRouters到我在App下挂载的路由上，例如：假设getRouterList是处理后台传过来的结构并且添加到Router上的方法，那我们应该这么使用：==

let routerlist=[]
　　//这里是调用router.js里的方法拿到动态路由
    Vue.prototype.ROUTER.getRouterList(function(data){
        routerlist=data
    })
　　//动态添加路由
    router.addRoutes(routerlist)
