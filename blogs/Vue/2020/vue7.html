# 本文从事件修饰符、按键修饰符和系统修饰符的使用以及使用他们需要注意的事项来记录。

理解：修饰符就是对绑定事件的一些修饰API来达到一个方便编程人员使用的工具。

# 事件修饰符

那既然是事件修饰符，就类似一些click、submit、scroll等等。对这些修饰来达成一定的目的，下面就是几个修饰符：

<!-- 阻止冒泡事件 -->
    <div class="box">
      <div @click="fatherBoo" class="boo">
        <button @click.stop="sonBoo">阻止冒泡</button>
      </div>
    </div>

# 按键修饰符和系统修饰符

这俩我就放一块说了,因为我感觉他就是一回事啊.....其实就是抓取用户对应的按键操作,要是匹配上就触发事件.

我先说这俩货长啥样:

<input @keyup.ctrl.67="copyThat"  v-model="cunshang"  type="text">
1

那都有啥呢?

  • .enter
  • .tab
  • .delete (捕获“删除”和“退格”键)
  • .esc
  • .space
  • .up
  • .down
  • .left
  • .right
  • .ctrl
  • .alt
  • .shift
  • .meta

上面这些修饰符简单易懂,身为键盘侠的你应该在你的武器上都能找到吧?除此之外官网还说了一堆按键码相关的事,我总结一下是这么个意思:"你要是非要犟用按键码也成,但是不建议你这么用,因为不同系统会出问题"

还有几个鼠标的:

  • .left
  • .right
  • .middle

这地方也没啥例子可举,但是需要注意几个地方,都是官方文档说的:

请注意修饰键与常规按键不同，在和 keyup 事件一起用时，事件触发时修饰键必须处于按下状态。换句话说，只有在按住 ctrl 的情况下释放其它按键，才能触发 keyup.ctrl。而单单释放 ctrl 也不会触发事件。如果你想要这样的行为，请为 ctrl 换用 keyCode：keyup.17。

最后还有一个修饰符,这个修饰符我再react里也遇见过:".exact"

.exact 修饰符允许你控制由精确的系统修饰符组合触发的事件。

<!-- 即使 Alt 或 Shift 被一同按下时也会触发 -->
<button v-on:click.ctrl="onClick">A</button>

<!-- 有且只有 Ctrl 被按下的时候才触发 -->
<button v-on:click.ctrl.exact="onCtrlClick">A</button>

<!-- 没有任何系统修饰符被按下的时候才触发 -->
<button v-on:click.exact="onClick">A</button>

# 事件修饰符注意!!!

我们可以注意到事件修饰符每个都有自己的作用,这个时候就可以串联使用来达到最终目的,举个栗子:

<a v-on:click.stop.prevent="doThat"></a>
1

那都啥不能一块用呢,官方给了两句话:

使用修饰符时，顺序很重要；相应的代码会以同样的顺序产生。因此，用 v-on:click.prevent.self 会阻止所有的点击，而 v-on:click.self.prevent 只会阻止对元素自身的点击。

不要把 .passive 和 .prevent 一起使用，因为 .prevent 将会被忽略，同时浏览器可能会向你展示一个警告。请记住，.passive 会告诉浏览器你不想阻止事件的默认行为。

第一句话我感觉是逻辑关系,但是没太理解透,目前是这么理解的:

prevent阻止了自己的事件本身的默认行为,而self阻止了内部的元素对自己行为的调用,如果先prevent再self就先阻止了自己的默认行为,相当于内部的也阻止了,内部点击也会因为self而导致所有的点击都失效;要是先self阻止的是内部的事件,再prevent的目标是自己??

现在我只能理解到这个程度,若是有大佬评论区教我一波

第二句话说的是这两个不能一起用,否则.prevent会失效,我在网上找了一定的资料,原因大致如下:

之前的滚动事件是当滚动结束的时候才会触发事件,就是因为为了保护浏览器有一个操作在一直屏蔽默认的事件,再出现了.passive做了一定的优化之后,这个就给阻止事件就被解除了,如果还要给他加就会报错.
