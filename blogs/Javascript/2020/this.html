# this的使用以及call、bind、apply

之前一直都使用this，但是这几天发现我对this理解还是太浅层，一些场景的使用也不是很灵活，写此文章来重新学习一遍。

# this的指向详情

this指向哪在定于这个函数的时候是不确定的，只有当函数执行的时候才能确定这个this到底指向的是谁，实际上this的最终指向是调用它的那个对象

大部分时候this指向的就是最终调用他的那个对象，但是还有其他的情况，下面是对这些情况的补充

情况1： 如果一个函数中有this，但是它没有被上一级的对象调用，那么this指向的就是window，那这里要是使用的是js严格模式，那this指向的不是window，而是undefined。

    function name()
    {
        let name1 = "wangjinshan"
        console.log("name====",this.name1);
        console.log("this=====",this);
    }
    name()

这个地方就很有意思，因为我是用wxz来调用b的fn，按理说this应该指向wxz，打印的结果应该是wangjinshan，但是打印的是wangxingze，证明指向的是b，可是上面说this指向的是最终指向呀，所以我们就分析上面的这三种情况：

  • 情况1：如果一个函数中有this，但是它没有被上一级的对象所调用，那么this指向的就是window。
  • 情况二：如果一个函数中有this，这个函数有被上一级的对象所调用，那么this指向的就是上一级的对象。
  • 情况三：如果一个函数中有this，这个函数中包含多个对象，尽管这个函数是被最外层的对象所调用，this指向的也只是它上一级的对象。

如果返回值是一个对象，那么this指向的就是那个返回的对象，如果返回值不是一个对象那么this还是指向函数的实例。

还有一点就是虽然null也是对象，但是在这里this还是指向那个函数的实例，因为null比较特殊。

function fn()  
{  
    this.user = 'wangxingze';  
    return null;
}
var a = new fn;  
console.log(a.user); //wangxingze
