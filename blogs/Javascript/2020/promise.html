详细解析Promise

王金山 2020-08-28 Javascript

# 认为对promise的理解太泛泛了，只知怎么用，不知为何，今天特做详细解析。

# 什么是Promise？他干什么了？

ES6提供了Promise对象，他是异步的一种实现方法，听名字嘛，Promise（承诺）就是答应了一件事，但是我做不做不一定，通过这种方式来实现了异步。Promise本身是一个构造函数，用于生成Promise实例，用new来生成。

# 为什么要出Promise？

按我理解就是为了偷懒，穿透的异步回调可能会因为业务逻辑的复杂，代码要写的很嵌套，就是“回调地狱”。

所以出现了Promise的链式编程来解决这个问题。给我的感觉就是一切皆可点

看看他是怎么用的哈：

        Promise.resolve().then(() => {
        <!--注意这步是同步的哈，点then之后才是异步操作-->
            console.log(6)
        }).then(() => {
            console.log(7)
        });

# Promise是怎么个原理

promise对象的状态有三种：pending(等待态),resolved(成功态),rejected(失败态)，只能从等待态转化成成功态或失败态；

那针对这几种状态我举个例子：

假设有一天小美和小明在大海里游泳，小美突然被海草缠住了脚（注意这个时候小美被缠住脚的事件已经发生了），这个时候小明说“你在这不要动，我去找人来救你”。这个时候小明给小美了一个承诺（Promise），这个时候的状态就是等待态（pending），这个时候就有两种可能了，小明带人来了（成功态resolved），小明跑了没管小美（失败态rejected）。我们可以看到在这个事中，只能事从pending到resolved或者到rejected的过程，他是不可逆的。

上面举了这个例子来说Promise的原理是啥，那我们现在想个场景，这个场景体现出了Promise的优势。

假设我们接着上面的故事讲，小美说：“我的明~~，你要是带人来救我，我就嫁给你！！”（那这个时候又产生了一个Promise），那小明听到了就很激动啊，回复到：“我的美~~~，你要是嫁给我，你肯定会是天下最幸福的女人”（又产生了一个Promise）。。。。。。。

上面这个故事就说到这，要是一直说我能写到地球毁灭了，反正意思就是可能会出现回调“套娃”的情况，所以这个时候肯定.then的写法比嵌套的写法要好太多太多了。

同时这个东西的实现也就是这么个思路。

# 手撕一个Promise

废话不多说，不懂就给我实现它！！！！

这个思路也是在网上找的，通过这个思路我也找到了编写的灵感。

  1. 最开始肯定是写个函数（就好比这俩人游泳去了），传入一个同步执行的方法家叫execuotor（就好比小美被海草缠住脚）
  2. 用一个字段（status）来记录Promise的状态，默认是“pending”，成功是“resolved”，失败是“rejected”，重写resolved和rejected方法，执行这两个方法时把status改变，以便以后then方法调用。
  3. 执行then方法时，依据status字段来执行成功方法或者失败方法。
function Promise(executor) {
    let self = this;
    self.status = 'pending'
    self.value = undefined
    self.reason = undefined

    function resolve(value) {
        if (self.status == 'pending') {
            self.value = value;
            self.status = 'resolved'
        }
    }
    function reject(reason) {
        if (self.status == 'pending') {
            self.reason = reason
            self.status = 'rejected'
        }
    }

    try {
        executor(resolve, reject)
    }
    catch{
        reject(e);
    }
}
Promise.prototype.then = function (onFulfilled, onRejected) {
    let self = this
    if(self.status == 'resolved')
    {
        onFulfilled(self.value)
    }
    if(self.status == 'rejected'){
        onRejected(self.reason)
    }
}

module.exports = Promise
