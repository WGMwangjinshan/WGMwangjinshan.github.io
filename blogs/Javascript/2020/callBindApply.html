call、bind和apply

王金山 2020-09-28 Javascript

# 写在前面

如果你还没看过我的《this的使用（个人理解）》，先去看那边回来再理解这篇文章。

# 三者相同的地方：

我们通过对this的剖析，已经知道这个东西在什么时候指向哪了，但是我们有时候吧不想this按他自己的想法指，要修改他就要更改this的指向，这个时候就有了call、bind和apply：

  • 都是用来改变函数的this对象的指向的。
  • 第一个参数都是this要指向的对象。
  • 都可以利用后续参数传参。

# 三者的区别是啥？都咋用的？

    }

那假如，现在想让Dilraba来调用wjs的do方法呢？













 
 
 

 let wjs = {
        name:'wangjinshan',
        age:'22',
        do:function()
        {
            console.log("我是",this.name,"今年",this.age,"岁");
        }
    }
    let Dilraba = {
        name:'Dilraba',
        age:'26',
    }
    wjs.do.call(Dilraba);//call的方法，立刻执行
    wjs.do.apply(Dilraba);//apply的方法，立刻执行
    wjs.do.bind(Dilraba)();//bind的方法，需要调用执行

call、apply与bind的区别

如果直接写xw.say.bind(xh)是不会有任何结果的，看到区别了吗？call和apply都是对函数的直接调用，而bind方法返回的仍然是一个函数，因此后面还需要()来进行调用才可以。 那么call和apply有什么区别呢？我们把例子稍微改写一下。

# 那call和apply又有什么区别呢？

看代码：













 
 

    let wjs = {
        name:'wangjinshan',
        age:'22',
        like:function(lover,aihao)
        {
            console.log("我是",this.name,"今年",this.age,"岁，我喜欢",lover,"，我爱好",aihao);
        }
    }
    let Dilraba = {
        name:'Dilraba',
        age:'26',
    }
    wjs.like.call(Dilraba,"王金山","吃")//call的传参
    wjs.like.apply(Dilraba,['王金山','吃'])//apply的传参

call、apply与bind的区别

看到区别了吗，call后面的参数与say方法中是一一对应的，而apply的第二个参数是一个数组，数组中的元素是和say方法中一一对应的，这就是两者最大的区别。 那么bind怎么传参呢？它可以像call那样传参。

1、

wjs.like.bind(Dilraba,"王金山","吃")()
1

2、

wjs.like.bind(Dilraba)("王金山","吃")
1

但是我们注意，bind返回的还是一个函数，所以我们可以在调用的时候再传参的嘛！！！！！！！
