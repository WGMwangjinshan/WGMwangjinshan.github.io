# 今天看《CSS世界》，发现BFC理解的还是肤浅，查缺补漏之BFC篇

# BFC官方定义

BFC(Block formatting context)直译为"块级格式化上下文"。它是一个独立的渲染区域，只有Block-level box参与， 它规定了内部的Block-level Box如何布局，并且与这个区域外部毫不相干。

看！这是多么官方的一句话（ps：我就不信这玩意不懂的人看了就能懂），第一次看这句话的时候我有两点点疑点：

  • “块级格式化上下文”是什么？
  • Block-level box是什么？

在说上面的这些疑惑和真正理解BFC之前，我们先理解两个概念：Box、Formatting Context

Box：css布局的基本单位

那说这个Box就是盒子，就咱们天天说的这放个盒子、那放个盒子的盒子。就可以这么说前端页面就是一个又一个的盒子配合组成的。一个盒子的元素属性和display属性，决定了这个盒子是个什么盒子；不同的盒子也就会参加不同的Formatting Context（就是一个有自己渲染文档规则的容器），因此盒子里的元素会用不同的方式渲染，那让我们看看有哪些类型的盒子：

  • block-level box：display属性为block，list-block、table的元素，会成为block-level box。参与的block fomatting context（BFC）

这个地方解决了我们一个疑问：Block-level box是什么？

  • inline-level box:display 属性为 inline, inline-block, inline-table 的元素，会生成 inline-level box。并且参与 inline formatting context（IFC）；
  • 其实还有种盒子（run-in box）,目前很少浏览器会支持这种盒子模型，我也没遇见过，所以这个地方我不做学习记录。

Formatting Context

上面我们说了，Formatting Context就是一个有自己渲染文档规则的容器，我感觉这句话很好理解了，再扩展一点就是：它是页面种的一块渲染区域，有自己的渲染规则，它自己决定了他子元素将如何定位，以及和其他元素的关系和相互作用。那我们上面说的BFC和IFC就是最常见的两种 Formatting context。

# BFC怎么用

这个部分咱们就来说说BFC是怎么定义自己的渲染规范且怎么才能让一个盒子成为一个BFC的

# 咋创建BFC？

就满足下面这些条件之一，就是一个BFC：

  • float的值不是none。
  • position的值不是static或者relative。
  • display的值是inline-block、table-cell、flex、table-caption或者inline-flex
  • overflow的值不是visible

# 结尾

这个时候我们都清楚了怎么触发BFC，和他都能干什么，这个时候我们回到刚开始那个问题：“块级格式化上下文”是什么？

其实按照我的理解就是不会影响周边任何其他元素，也不会被其他元素影响的一个独立容器。

因为BFC内部的元素和外部的元素绝对不会互相影响，因此， 当BFC外部存在浮动时，它不应该影响BFC内部Box的布局，BFC会通过变窄，而不与浮动有重叠。同样的，当BFC内部有浮动时，为了不影响外部元素的布局，BFC计算高度时会包括浮动的高度。避免margin重叠也是这样的一个道理。
